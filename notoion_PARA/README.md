# 노션 PARA 기반 세컨브레인 자동화 시스템 사용 설명서 🧠

본 프로젝트는 티아고 포르테(Tiago Forte)의 **PARA 프레임워크**와 **CODE 방법론**을 기반으로 구축된 지식 관리 자동화 파이프라인입니다. 
당신의 아이디어를 수집(Capture)하기만 하면, AI가 알아서 분류(Organize), 핵심 요약(Distill), 그리고 기획안과 주간 리뷰로 표현(Express)합니다.

---

## 🏗 시스템 아키텍처 및 원리

사용자가 **노션 Inbox** 페이지에 기사 내용이나 아이디어를 복사하여 붙여넣으면, 저장된 정보는 아래의 4단계를 거쳐 활용 가능한 자산으로 탈바꿈합니다.

1.  **Capture (수집)**: 모바일 앱, 웹 클리퍼, 직접 입력 등을 통해 모든 아이디어를 `Inbox` 데이터베이스에 무작위로 수집합니다.
2.  **Classify (분류)**: AI가 내용을 읽고 프로젝트(Project), 영역(Area), 자료(Resource) 등 PARA 유형을 결정하여 알맞은 노션 DB로 보내고 연관된 프로젝트와 릴레이션(Relation)을 맺습니다.
3.  **Distill (정제)**: '검토 필요' 상태의 지식들을 모아 AI가 L2/L4 깊이를 부여하고, 핵심 문장과 '실행 가능한 인사이트(Actionable Insights)'를 3줄 요약합니다.
4.  **Express (표현)**: 수집된 재료들을 기반으로 현재 진행 중인 프로젝트의 **AI 기획안(Project Brief)**을 생성하거나, 데이터들을 종합하여 **주간 리뷰(Weekly Review)** 리포트를 자동으로 노션에 작성합니다.

---

## 🛠 실행 전 준비 사항

본 폴더(`d:\안티그래비티\notoion_PARA`)에는 이미 코드가 완성되어 있습니다. 실행하려면 다음 환경이 갖추어져 있어야 합니다.

1.  **필수 프로그램**: Node.js가 설치되어 있어야 합니다. (현재 컴퓨터에 설치되어 있습니다)
2.  **환경 변수 (`.env`) 설정**:
    폴더 내에 `.env` 파일이 존재해야 하며, 파일 안에 다음 값들이 올바르게 입력되어 있어야 합니다. (이미 전부 세팅되어 있습니다.)
    *   `NOTION_TOKEN`: 노션 API 통합 시크릿 토큰
    *   `GEMINI_API_KEY`: 구글 제미나이 AI API 키
    *   `NOTION_PAGE_ID`: 워크스페이스 최상위 페이지 ID
    *   `NOTION_INBOX_DB_ID` ~ `NOTION_KNOWLEDGE_DB_ID`: 각 데이터베이스 ID

## 🚀 자동화 스크립트 실행 가이드

모든 스크립트는 `notoion_PARA` 폴더에서 명령 프롬프트(CMD) 또는 터미널을 열고, 아래 명령어들을 입력하여 실행합니다.

### 1단계: 수집품 AI 자동 분류 (Classify)
*   **명령어**: `node src/jobs/classifyInbox.js`
*   **언제 실행하나요?**: 하루 일과를 마치기 전 또는 생각날 때마다.
*   **결과**: `Inbox` DB에 있는 내용을 AI가 분석한 뒤, `Knowledge` DB로 깔끔하게 속성(프로젝트 등 관계 매핑 포함)을 채워서 옮기고 인박스를 비웁니다.

### 2단계: 핵심 지식 요약 및 정제 (Distill)
*   **명령어**: `node src/jobs/distillKnowledge.js`
*   **언제 실행하나요?**: 분류 작업이 끝난 후 이어서 실행합니다.
*   **결과**: 방금 막 `Knowledge DB`로 들어온 긴 글의 문서를 AI가 3줄 요약하고 '액션 아이템', '언제 다시 쓸 수 있는지(Next Use)' 등의 인사이트를 추출합니다.

### 3단계: 특정 프로젝트 기획안 초안 자동 작성 (Express)
*   **명령어**: `node src/jobs/expressProjectBrief.js`
*   **언제 실행하나요?**: 새로운 기획서를 작성해야 하거나 아이디어가 고갈되었을 때.
*   **결과**: `Projects` DB에 있는 '진행 중(Active)' 프로젝트들을 검색한 다음, 그 프로젝트와 연관되어 수집된 지식들을 전부 끌어모아 **훌륭한 기획 초안(Project Brief)**을 작성해 해당 프로젝트 페이지 내부에 추가합니다.

### 4단계: 시스템 주간 리뷰 (Weekly Review)
*   **명령어**: `node src/jobs/weeklyReview.js`
*   **언제 실행하나요?**: 매주 금요일 퇴근 전이나 일요일 저녁.
*   **결과**: 남아있는 인박스 수, 현재 진행 중인 프로젝트 현황, 이번 주에 수집된 핵심 인사이트(L2/L4)를 기반으로 **"다음 주에 집중해야 할 Top 3 우선순위"**를 도출해 내는 주간 리포트를 노션 최상위 페이지 최상단에 발행합니다.

## 🖥 노션 대시보드(Home) 최적화 방법

가장 편하게 PARA 시스템을 운영하기 위해, 여러분의 노션 `P.A.R.A` 페이지(또는 템플릿 메인 화면)에 다음 **링크된 데이터베이스 보기(Linked View of Database)**를 구성해두는 것을 추천합니다.

1.  **🚀 빠른 캡처 존**:
    *   `Inbox` DB를 리스트 뷰로 띄웁니다.
    *   **필터**: `Processed`가 체크 해제된 것만 보이도록 설정. (분류 스크립트가 돌면 여기서 자동으로 사라집니다!)
2.  **🏃 진행 중인 프로젝트**:
    *   `Projects` DB를 갤러리 뷰(안에 내용물 보이게 렌더링)로 띄웁니다.
    *   **필터**: `Status`가 `Active`인 프로젝트만 보이게 합니다.
3.  **💎 내 두 번째 뇌 (정제된 지식)**:
    *   `Knowledge` DB를 보드 뷰(Kanban)로 띄우고, 열(Column) 기준을 `PARA Type`으로 정렬합니다.
    *   **필터**: `Review Needed`가 No이고, `Layer`에 값이 있는 항목만 모아보세요. 진정한 나만의 핵심 자료실이 완성됩니다.
